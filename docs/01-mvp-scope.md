# MVP Scope

## Objetivo del MVP

El MVP de SaleOps debe validar el flujo ecommerce-first completo: un cliente descubre productos, realiza un pedido y el negocio lo gestiona desde el panel interno con inventario, ventas, reportes basicos e IA interna.

El MVP no busca cubrir todos los procesos de una empresa. Debe demostrar una operacion comercial moderna, segura y mantenible sin convertirse en marketplace, ERP o plataforma multiempresa.

## Criterios de inclusion

Una funcionalidad entra al MVP si cumple al menos una de estas condiciones:

- ayuda a completar una compra;
- ayuda al negocio a cumplir un pedido;
- protege consistencia de inventario, pago o cliente;
- permite medir ventas basicas;
- demuestra el valor operativo de SaleOps.

## Criterios de exclusion

Una funcionalidad se posterga si:

- requiere integracion externa compleja;
- cambia el producto hacia marketplace o ERP;
- agrega multitenancy antes de validar una instalacion;
- mejora marketing pero no el flujo operativo central;
- aumenta riesgo de seguridad sin aportar valor al MVP.

## Recommended Improvements

- Crear una lista de historias de usuario por cada modulo incluido.
- Definir criterios de aceptacion para checkout, pedidos e inventario antes de programar.
- Mantener un backlog separado para ideas futuras y evitar que contaminen el MVP.
